# FileServer 项目设计文档

## 1. 项目介绍

### 1.1 项目背景

随着互联网应用对文件存储和传输需求不断增加，传统基于阻塞IO模型的文件服务器在高并发场景下存在性能瓶颈，例如：

- 单连接阻塞导致线程资源浪费；
- 大文件传输过程中容易受到网络波动影响；
- 文件元数据管理与数据存储耦合，不利于扩展；
- 服务规模扩大后无法实现服务水平扩展。

本项目设计并实现一个轻量级分布式文件管理服务，采用 C++11 开发，基于 muduo 网络库实现高并发网络通信，通过主从 Reactor 多线程模型处理客户端连接。

系统支持：

- 用户注册与认证；
- 文件上传、下载、删除；
- 文件元数据管理；
- 大文件分块传输；
- 断点续传；
- 分布式服务注册与发现。

# 2. 系统整体架构

系统采用微服务架构，将文件管理功能拆分为多个独立服务。

```
                         Client

                           |
                           |
                    TCP长连接通信

                           |

                 +----------------+
                 | Gateway Server |
                 |  接入层服务     |
                 +----------------+

                           |

                     mprpc调用

                           |

                 +----------------+
                 | ZooKeeper      |
                 | 服务注册中心   |
                 +----------------+

              /              |              \

             /               |               \


       UserService      FileService     StorageService

           |                |                |

        MySQL           MySQL          文件磁盘存储
```

# 3. Reactor线程模型设计

服务端采用：

> 主从 Reactor 多线程模型

## 3.1 架构

```
                 Main Reactor

                    |
                    |
             accept新客户端连接

                    |

        --------------------------------

        |              |               |

        v              v               v


    Sub Reactor1   Sub Reactor2   Sub Reactor3

        |              |               |

        |              |               |

     IO事件处理    IO事件处理    IO事件处理


        \              |              /

                 Task Queue

                      |

              Worker ThreadPool

                      |

          ----------------------

          |         |          |

       User     File      Storage

       Service Service   Service
```

------

# 3.2 Main Reactor职责

主线程只负责：

- 监听端口；
- 接收客户端连接；
- 创建TCP连接；
- 将连接分配给Sub Reactor。

流程：

```
Client connect

      |

Main Reactor

      |

accept()

      |

选择Sub Reactor

      |

注册Channel
```

------

# 3.3 Sub Reactor职责

每个Sub Reactor运行于独立线程。

负责：

- epoll事件监听；
- socket读写；
- TCP连接维护；
- 网络buffer管理。

数据处理流程：

```
TCP数据到达

     |

Sub Reactor

     |

读取Buffer

     |

JSON协议解析

     |

生成业务任务

     |

投递线程池
```

------

# 3.4 Worker线程池

业务线程负责：

- MySQL查询；
- 文件IO；
- RPC调用。

避免：

```
Reactor线程

      |

磁盘IO

      |

阻塞EventLoop
```

导致其它客户端无法响应。

------

## 5.2 文件元数据表 file_info

```
CREATE TABLE file_info
(
    id INT PRIMARY KEY AUTO_INCREMENT,

    user_id INT,

    filename VARCHAR(255),

    filepath VARCHAR(512),

    filesize BIGINT,

    upload_size BIGINT,

    md5 VARCHAR(64),

    status INT,

    create_time DATETIME
);
```

字段说明：

| 字段        | 说明           |
| ----------- | -------------- |
| filename    | 文件名称       |
| filepath    | 服务器存储路径 |
| filesize    | 文件总大小     |
| upload_size | 已经上传大小   |
| md5         | 文件唯一标识   |
| status      | 上传状态       |

status：

```
0 上传中

1 上传完成

2 删除
```

# 项目总结

FileServer 是一个基于 C++11 实现的分布式文件管理系统。

系统采用 muduo 主从 Reactor 多线程模型实现高并发 TCP 网络通信，通过自定义 JSON 私有协议解决 TCP 粘包半包问题；利用 mprpc 和 ZooKeeper 实现服务间通信以及服务注册发现；通过 MySQL 管理用户和文件元数据，实现大文件分块上传、下载以及断点续传功能。
